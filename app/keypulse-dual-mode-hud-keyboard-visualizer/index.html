See it in action

One extension. Two ways to visualize your keyboard.

Switch between a detailed full keyboard and a minimal floating HUD, depending on how much visual context you need.

Full Keyboard Mode

See every key as you press it.

The full keyboard view maps physical keys and keeps simultaneous presses highlighted together, making combinations such as Ctrl + Shift + A easy to demonstrate and follow.

Compact HUD Mode

Keyboard visualization, without leaving the page.

Keep KeyPulse available as a small floating HUD while you browse, present, record, or demonstrate shortcuts. Collapse or expand it with the side handle whenever you need more or less screen space.

Built for clarity

Useful when the keyboard is part of the story.

KeyPulse keeps keyboard input visible without turning your workflow into a recording setup or sending your activity to a remote service.

01

Physical-key mapping

Uses physical keyboard key codes so the on-screen visualization follows the keys you actually press.

02

Simultaneous shortcuts

Multiple held keys stay highlighted together, making complex keyboard shortcuts easier to see in real time.

03

Left and right modifiers

Distinguishes left and right Shift, Ctrl, Alt, and Meta keys in full keyboard mode.

04

Floating on-page interface

The visualizer appears directly on ordinary HTTP and HTTPS pages, keeping it close to the content you are presenting.

05

Remembered position and mode

Local extension storage remembers widget visibility, position, and the selected display mode.

06

Focus-safe highlighting

Active highlights reset when the page loses focus, helping prevent keys from appearing stuck on screen.

Privacy by design

Your keystrokes stay local.

KeyPulse is built to visualize keyboard activity in the browser, not to collect it.

No network requests or backend

KeyPulse does not send keyboard activity to a server and does not depend on a remote backend.

No telemetry, analytics, or advertising

The extension does not use analytics services, advertising SDKs, or behavioral tracking.

No key history

Keyboard events are used in page memory to update the visualizer. A history of typed keys is not stored or transmitted.

Password fields are ignored

KeyPulse does not visualize keyboard activity while typing into password input fields.
