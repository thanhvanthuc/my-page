About the work

Small products can still feel deeply considered.

I believe useful software does not need to be oversized, overloaded, or difficult to understand. A focused product can create real value when its purpose is clear and every detail supports that purpose.

Independent by nature

Working independently creates room to explore ideas quickly, challenge assumptions, and shape products around real usability instead of process for its own sake. It also makes it possible to stay close to the entire product lifecycle — from the first concept to the interface, technical behavior, support, and refinement.

Utility over noise

Not every product needs an enormous feature list. I prefer tools that are easy to understand, fast to access, and confident about what they are designed to do. Good scope is a feature: it helps a product stay reliable, understandable, and enjoyable over time.

Design is part of the function

Visual polish is not decoration added at the end. Layout, hierarchy, motion, spacing, feedback, and accessibility all affect whether a product feels natural or frustrating. The interface should quietly guide the user instead of competing for attention.

Trust should be designed in

Privacy-conscious choices, clear communication, and predictable behavior are part of the product experience. When possible, I favor local processing, minimal permissions, transparent explanations, and collecting less rather than collecting everything by default.

How I work

From vague problem to focused product.

The exact process changes from project to project, but the underlying approach stays consistent: understand the need, simplify the solution, build carefully, then refine through real use.

01 — Understand

Find the real friction.

Before expanding into features, define the actual problem, the user context, and what a successful experience should feel like.

→
02 — Simplify

Remove everything that does not help.

Reduce steps, visual noise, and unnecessary choices until the core interaction becomes obvious and easy to explain.

→
03 — Build

Create the smallest complete experience.

Focus first on reliability, coherent behavior, and a strong end-to-end experience instead of chasing a long feature checklist.

→
04 — Refine

Polish where real use exposes friction.

Improve interactions, edge cases, layout, performance, and communication based on how the product behaves outside the first design.

→
