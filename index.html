About the work

Small products can still feel deeply considered.

Independent by nature

Utility over noise

Not every product needs an enormous feature list. Good scope helps software stay reliable, understandable, and enjoyable over time.

Design is part of the function

How I work

From vague problem to focused product.

The exact process changes from project to project, but the underlying approach stays consistent: understand the need, simplify the solution, build carefully, then refine through real use.

01

Understand the real friction.

Define the actual problem, the user context, and what a successful experience should feel like before expanding into features.

02

Simplify until the core is obvious.

Remove unnecessary steps, visual noise, and choices until the essential interaction becomes clear and easy to explain.

03

Build the smallest complete experience.

Prioritize reliability, coherent behavior, and a strong end-to-end flow instead of chasing a long feature checklist.

04

Refine where real use exposes friction.

Improve interactions, layout, performance, communication, and edge cases based on how the product behaves outside the first design.
